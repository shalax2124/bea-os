1. Create src/middleware.ts — Clerk auth middleware:
```typescript
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'

const isPublicRoute = createRouteMatcher([
  '/sign-in(.*)',
  '/sign-up(.*)',
])

export default clerkMiddleware(async (auth, request) => {
  if (!isPublicRoute(request)) {
    await auth.protect()
  }
})

export const config = {
  matcher: [
    /*
     * Match all request paths except:
     * - _next/static (static files)
     * - _next/image (image optimization)
     * - favicon.ico (favicon)
     * - public folder assets
     */
    '/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)',
  ],
}
```

2. Create src/app/sign-in/[[...sign-in]]/page.tsx — Clerk's hosted sign-in component:

Key details:
- clerkMiddleware with createRouteMatcher protects all routes except /sign-in and /sign-up
- auth.protect() redirects unauthenticated users to the sign-in page
- The sign-in page uses Clerk's built-in SignIn component — no custom form needed
- The [[...sign-in]] catch-all route pattern is required by Clerk for multi-step auth flows
